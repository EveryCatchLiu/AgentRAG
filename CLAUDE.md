# CLAUDE.md

RAG 应用，具有聊天（默认）和文件导入界面。通过环境变量配置，没有管理界面。

## 技术栈

- 前端：React + Vite + Tailwind + shadcn/ui
- 后端：Python + FastAPI
- 数据库：Supabase（Postgres、pgvector、认证、存储、实时）
- LLM：OpenAI（模块 1）、OpenRouter（模块 2+）
- 可观测性：LangSmith

## 规则

- Python 后端必须使用 `venv` 虚拟环境
- 不使用 LangChain，不使用 LangGraph - 仅使用原始 SDK 调用
- 使用 Pydantic 进行结构化 LLM 输出
- 所有表都需要行级安全性 - 用户只能看到自己的数据
- 通过 SSE 进行流式聊天回应
- 使用 Supabase Realtime 进行导入状态更新
- 模块 2+ 使用无状态 Completions - 自己存储和发送聊天历史
- 导入仅限手动文件上传 - 没有连接器或自动化管道

## 规划

- 将所有计划保存到 `.agent/plans/` 文件夹
- 命名惯例：`{序号}.{计划名称}.md`（例如 `1.认证设置.md`、`2.文件导入.md`）
- 计划应足够详细，可以无歧义地执行
- 计划中的每个任务必须包含至少一个验证测试来确认其正常工作
- 评估复杂度和一次性可行性 - 代理能否真实地一次完成？
- 在每个计划顶部包含复杂度指标：
  - ✅ **简单** - 可一次执行，低风险
  - ⚠️ **中等** - 可能需要迭代，有一定复杂性
  - 🔴 **复杂** - 执行前拆分为子计划

## 开发流程

1. **规划** - 创建详细计划并保存到 `.agent/plans/`
2. **构建** - 执行计划来实现功能
3. **验证** - 测试并验证实现是否正确工作。适用时通过适当的 MCP 进行浏览器测试
4. **迭代** - 修复验证中发现的任何问题
